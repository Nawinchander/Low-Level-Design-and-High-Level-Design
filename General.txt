In system design interviews, High-Level Design (HLD) and Low-Level Design (LLD) are two different layers of problem solving:

HLD = Architecture level
Focuses on how major components interact
LLD = Code level
Focuses on how classes/modules/functions are implemented

Think of it like building a food delivery app:

HLD → Services like User Service, Order Service, Payment Service, DB, Cache
LLD → Classes like Order, PaymentProcessor, RestaurantManager


| Aspect      |                  HLD |                  LLD |
| ----------- | -------------------: | -------------------: |
| Focus       |         Architecture |          Code Design |
| Level       |                Macro |                Micro |
| Asked In    |         Senior / Mid |          SDE1 / SDE2 |
| Main Topics |   DB, Cache, Scaling |         Classes, OOP |
| Output      | Architecture Diagram | Class Diagram / Code |


FAANG Interview Answer Strategy
If interviewer asks HLD:

Say:

“I’ll define requirements, estimate scale, design APIs, choose DB/cache, then discuss bottlenecks and scaling.”

This gives structure.

If interviewer asks LLD:

Say:

“I’ll identify entities, define relationships, apply SOLID principles, and design extensible classes.”

This shows maturity.
